# もぐもぐウォーク iOS アプリ 要件定義

## 1. アプリ概要

| 項目 | 内容 |
|------|------|
| アプリ名 | もぐもぐウォーク |
| 目的 | 矯正トレーニング中の歩数計測 |
| ターゲット | 矯正トレーニングを行う子供（保護者が設定・利用） |
| プラットフォーム | iOS 16.0 以上 / iPhone |
| 配布方法 | TestFlight |
| 言語 | 日本語のみ |

### Web版からの移行理由

| 課題 | Web版（現行） | iOS版（解決後） |
|------|------------|------------|
| バックグラウンド再生 | 画面オフでメトロノームが停止 | Background Audio で継続再生 |
| 歩数精度 | 自前アルゴリズム（閾値1.5、間隔300ms）で低精度 | CMPedometer（Appleモーションコプロセッサ）で高精度 |
| センサー権限 | 毎回ユーザー操作が必要 | 初回のみ許可 |

---

## 2. 機能要件

### 2-1. 歩数カウント

**要件**：
- 「はじめる」ボタンタップで `CMPedometer.startUpdates()` を開始
- 計測中はリアルタイムで歩数を画面に表示（整数）
- 「とめる」ボタンタップで停止 → 歩数をセッションに保存
- CMPedometer 利用不可の端末ではエラーメッセージを表示

**権限**：
- `NSMotionUsageDescription` を Info.plist に記述
- 初回起動時に CMMotionActivityManager の権限ダイアログが表示される

### 2-2. メトロノーム

**要件**：
- BPM: **120**（固定、変更不可）
- 音: 短いクリック音（50ms、1200Hz サイン波 + 指数減衰エンベロープ）
- アプリ起動時は **OFF**
- ボタンタップで ON/OFF トグル
- 「とめる」タップ時にメトロノームも自動停止
- 画面オフ・バックグラウンド時も継続再生

**技術要件**：
- AVAudioEngine + スケジューリングで高精度なタイミングを実現
- AVAudioSession カテゴリ: `.playback`
- Background Modes: `Audio, AirPlay, and Picture in Picture` を有効化

### 2-3. セッション管理

**要件**：
- 1日最大 **6セッション**
- 6回に達したら「はじめる」ボタンを非活性化
- 各セッションに保存する情報:

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `slot` | Int | セッション番号（1〜6の連番） |
| `steps` | Int | 歩数（整数） |
| `time` | String | 停止時刻（`HH:MM`、24時間形式） |

- セッション一覧を画面に表示
- 個別削除可能（削除確認あり: "消す" / "やめる"）
- 削除後は slot を 1 から連番に詰め直す（例: 1,2,3 → 2を削除 → 1,2）
- 日付が変わるとセッションを自動リセット（アプリ起動・フォアグラウンド復帰時に判定）

### 2-4. データ永続化

**要件**：
- UserDefaults に JSON 形式で保存
- キー: `mogumogu-walk-training`

**保存形式**：
```json
{
  "date": "2026-05-23",
  "sessions": [
    { "slot": 1, "steps": 150, "time": "09:30" },
    { "slot": 2, "steps": 200, "time": "10:45" }
  ]
}
```

## 3. 画面仕様

### メイン画面（単一画面）

```
┌─────────────────────────────┐
│                             │
│          150 歩             │  ← 大きな歩数表示（紫、等幅フォント）
│                             │
│  ┌──────────┐ ┌──────────┐  │
│  │🐾 はじめる│ │🔕 メトロ  │  │  ← Idle 状態
│  └──────────┘ └──────────┘  │
│                             │
│  ┌──────────────────────┐   │  ← Counting 状態（はじめると切替）
│  │      ⏹ とめる         │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │  🔔 メトロノーム ON   │   │
│  └──────────────────────┘   │
│                             │
│  📋 今日の記録 (2/6)        │
│  ┌─────────────────────────┐ │
│  │ 🐾 09:30   150歩  [🗑]  │ │
│  │ 🐾 10:45   200歩  [🗑]  │ │
│  └─────────────────────────┘ │
└─────────────────────────────┘
```

### 状態遷移

```
Idle → [はじめる] → Counting → [とめる] → Idle（保存）
  │
  └→ CMPedometer未対応 → エラーメッセージ表示
```

### カラースキーム

| 要素 | 色 |
|------|-----|
| 歩数・アクセント | Purple |
| はじめるボタン | Green |
| とめるボタン | Red |
| メトロノームON | Orange |
| メトロノームOFF | Blue |
| 背景 | 薄紫（.purple.opacity(0.1) 相当） |

### UIコンポーネント

- 歩数表示: `Text` + `.monospacedDigit()` + 大フォント（system font）
- ボタン: `Button` + `RoundedRectangle` + `.fill` スタイル、角丸大（cornerRadius: 20）
- セッションリスト: `ScrollView + VStack`
- 削除確認: インライン切り替え（"消す" / "やめる"）
- デザイン方針: スマホ縦持ち前提、パステルカラー基調、アイコンはすべて絵文字

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 対応OS | iOS 16.0 以上 |
| 対応デバイス | iPhone（縦持ち固定） |
| 言語 | 日本語のみ |
| ダークモード | 対応（SwiftUI自動対応） |
| アクセシビリティ | Dynamic Type対応（SwiftUIデフォルト） |
| オフライン | 全機能オフライン動作（ネットワーク不要） |
| プライバシー | 歩数データは端末内のみ保存、外部送信なし |
| 配布 | TestFlight のみ（App Store 公開なし） |

---

## 5. 技術スタック

| 領域 | 技術 |
|------|------|
| 言語 | Swift 5.9+ |
| UI フレームワーク | SwiftUI |
| 歩数カウント | CoreMotion（CMPedometer） |
| 音声 | AVFoundation（AVAudioEngine + AVAudioPlayerNode） |
| データ永続化 | UserDefaults（Codable） |
| 最小ターゲット | iOS 16.0 |
| Xcode | 15.0 以上 |

---

## 6. Info.plist 必須エントリ

```xml
<!-- 歩数センサー利用の説明 -->
<key>NSMotionUsageDescription</key>
<string>歩数を計測するために動作センサーを使用します。</string>

<!-- 画面オフ時もメトロノーム再生を継続するため -->
<key>UIBackgroundModes</key>
<array>
  <string>audio</string>
</array>

<!-- 縦持ち固定 -->
<key>UISupportedInterfaceOrientations</key>
<array>
  <string>UIInterfaceOrientationPortrait</string>
</array>
```

---

## 7. スコープ外（本バージョンでは対応しない）

- HealthKit 連携（歩数データの同期）
- BPM 変更機能
- 複数日の記録閲覧・履歴機能
- ウィジェット
- Apple Watch 対応
- 通知・リマインダー
- iPad 対応
