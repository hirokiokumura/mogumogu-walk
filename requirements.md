# もぐもぐウォーク MVP 要件定義

## 1. 概要

矯正トレーニング中に、開始ボタンを押してから歩いた歩数をスマホブラウザで計測するWebアプリを開発する。

子供が使いやすいように、スマホで見やすく、パステルカラーを基調にしたかわいいUIとする。

## 2. MVPの目的

- iPhone Safariなどのスマホブラウザで歩数を計測できることを確認する
- 1日6回までのトレーニング記録を保存できること
- App Store公開なしで、GitHub Pagesから利用できること
- 子供が迷わず使えるシンプルなUIにすること

## 3. 対象ユーザー

- 矯正トレーニングを行う子供
- 子供のトレーニング状況を確認する保護者

## 3.1 対応環境

- iPhone Safari のみ対応
- Android は MVP では未対応

## 4. 技術スタック

### フロントエンド

- Next.js
- TypeScript
- Tailwind CSS
- PWA対応

### ホスティング

- GitHub Pages

### データ保存

- localStorage

### センサー

- DeviceMotionEvent
- 加速度センサーを利用して歩数を推定する

## 5. デプロイ方針

GitHub Pagesで公開するため、Next.jsは静的サイトとして出力する。

`next.config.js` では以下の設定を想定する。

```js
const nextConfig = {
  output: 'export',
  basePath: '/mogumogu-walk',
  assetPrefix: '/mogumogu-walk',
};

module.exports = nextConfig;
```

API Routes、Server Actions、サーバーサイドDBはMVPでは使用しない。

## 6. 主要機能

### 6.1 歩数計測

- ユーザーが「はじめる」ボタンを押すと歩数計測を開始する
- iPhone Safariでは、開始ボタン押下後にセンサー利用許可を求める
- 加速度センサーの値から歩行らしい揺れを検出し、歩数としてカウントする
- ユーザーが「とめる」ボタンを押すと計測を停止する
- 停止時に、その回の歩数を保存する
- 計測中は「はじめる」ボタンを非表示または無効化する
- 停止後、各回の記録に「この回をリセット」ボタンを表示し、押すとその回の記録を削除して再計測できる状態に戻す

### 6.2 1日6回の記録

- 1日あたり最大6回まで記録できる
- 今日の記録回数を `2 / 6 回` のように表示する
- 6回に到達したら、新規計測はできないようにする
- 日付が変わったら自動的に当日の記録を初期化する
- 手動リセットボタンで当日の全記録を削除できる（確認ダイアログを表示する）
- 各回ごとにも個別リセットが可能（「この回をリセット」ボタン）

### 6.3 localStorage保存

localStorageに日付単位で保存する。

キー名:

```ts
const STORAGE_KEY = 'mogumogu-walk-training';
```

保存データ例:

```json
{
  "date": "2026-05-21",
  "sessions": [
    {
      "slot": 1,
      "steps": 120,
      "time": "08:10"
    },
    {
      "slot": 2,
      "steps": 98,
      "time": "10:30"
    }
  ]
}
```

slot の採番ルール:

- 追加時は現在のセッション数 + 1 を割り当てる
- 個別リセット（削除）後は slot を 1 から連番に詰め直す
  - 例: slot 1, 2, 3 の状態で slot 2 を削除 → slot 1, 2 に採番し直す

### 6.4 メトロノーム

- メトロノーム音を再生できる
- 音源ファイルは使用せず、Web Audio API でプログラム的にクリック音を生成する
- ON/OFFを切り替えられる
- 初期状態は OFF とする（アプリ起動時に自動再生しない）
- MVPではBPMは固定値でよい
- 初期値は 120 BPM とする（500ms ごとにクリック音を鳴らす）
- 将来的にBPM変更UIを追加できる設計にする

### 6.5 PWA対応

- ホーム画面に追加できる
- アプリアイコンを設定する（絵文字ベースで生成する。デザイン素材なし）
- アプリ名は「もぐもぐウォーク」とする
- 一度読み込んだ後、オフラインでも動作する（Service Worker でキャッシュ）
  - localStorage・加速度センサーはオフラインでも動作するため、実現可能
  - 初回起動時のみネット接続が必要

## 7. UI要件

### 7.1 デザイン方針

- スマホ縦持ち前提
- 全体的にパステルカラー
- アイコンは絵文字を使う
- カードUI形式で記録を表示する
- 角丸を大きめにする
- ボタンは大きく押しやすくする
- 文字は子供でも読みやすいサイズにする

### 7.2 画面構成

#### ヘッダー

```text
🌈 もぐもぐウォーク
```

#### 今日の回数カード

```text
📅 今日のきろく

2 / 6 回
```

※ 合計歩数は表示しない。

#### 現在計測カード

```text
🚶 あるいてるよ！

👣 98歩
```

#### メトロノームカード

```text
🎵 メトロノーム

ON / OFF
```

#### 操作ボタン

```text
🟢 はじめる
🔴 とめる
🔄 リセット
```

#### 履歴カード

```text
📝 今日のきろく

👣 1回目
120歩
08:10
[この回をリセット]

👣 2回目
98歩
10:30
[この回をリセット]
```

### 7.3 表示しないもの

- 合計歩数は表示しない
- 複雑な分析グラフはMVPでは不要
- ログイン機能は不要

## 8. 歩数計測の注意点

Webブラウザの加速度センサーを利用した歩数計測は、iPhoneのヘルスケアと同等の精度にはならない。

MVPでは以下を前提とする。

- スマホを手に持つ、または一定の位置に持って歩く
- 短時間のトレーニング用途とする
- 歩数の厳密性よりも、継続しやすさを優先する
- 誤検知がある前提で、将来的に補正ロジックを改善する

## 9. iPhone Safari対応

iPhone Safariでは、ユーザー操作後にセンサー許可を取得する必要がある。

実装例:

```ts
async function requestMotionPermission() {
  if (
    typeof DeviceMotionEvent !== 'undefined' &&
    typeof (DeviceMotionEvent as any).requestPermission === 'function'
  ) {
    const permission = await (DeviceMotionEvent as any).requestPermission();
    return permission === 'granted';
  }

  return true;
}
```

## 10. 完了条件

MVPの完了条件は以下とする。

- GitHub Pagesで公開できている
- iPhone Safariで画面表示できる
- 開始ボタンでセンサー許可を取得できる
- 歩くと歩数が増える
- 停止ボタンで記録がlocalStorageに保存される
- 1日6回まで記録できる
- 日付変更時に当日記録がリセットされる
- メトロノームのON/OFFができる
- スマホで見やすいパステルカラーのカードUIになっている
